# 一笔留白：爽切反馈与魔性结算设计

日期：2026-07-18

## 1. 目标

本阶段建立游戏的两条核心产品链路：

1. 爽切反馈链：让每次成功切割产生清晰、锋利、解压的即时反馈，承担留存和重复游玩的核心价值。
2. 魔性结算链：用网络热梗评价、统一声音人格和递进式视听反馈形成记忆点，承担传播和品牌识别价值。

设计原则是“切割体验为主体，热梗结算为放大器”。结算不得遮挡最终一刀的释放过程，语音不得替代基础手感。

## 2. 本阶段范围

包含：

- 成功切割的视觉、音频、停顿、震动、位移和进度反馈。
- 从完成目标到结算页可操作的完整时间轴。
- 时间主导、生命限级的五档评价规则。
- 统一声音人格及五档情绪强度。
- 本关用时、剩余生命、挥墨次数和最佳纪录展示。
- 再来一次、下一关两个明确操作。

不包含：

- 切割次数限制或切割次数扣分。
- 在线排行榜、好友排名、账号系统和分享接口。
- 凹形反弹算法、真实曲线碰撞和新地图扩展。
- 直接复制其他游戏的语音、音乐或动画素材。

## 3. 产品定位

### 3.1 爽切负责留存

玩家应当因为“这一刀切得舒服”而继续游戏。切割次数不限，挥墨次数只作为趣味统计，不进入评分公式。系统不得一边鼓励玩家自由切割，一边因为多切而扣分。

### 3.2 热梗负责记忆和传播

“夯爆了”是最高等级的稀有奖励，不是每关固定台词。结算页天然适配竖屏截图和录屏：评价、关卡、用时、剩余生命和产品名称应在一个画面内成立。

## 4. 爽切反馈链

一次有效切割由六段组成，总时长约 300 至 500 毫秒。

### 4.1 起笔

- 起笔时播放轻微毛笔摩擦声。
- 切线从淡到浓聚集，不出现突兀的粗线跳变。
- 第一次操作负责解锁浏览器音频，不延迟首次切割判定。

### 4.2 划动

- 切线保持锋利、稳定和低延迟。
- 接近墨刃或墨铁封边时使用克制的朱红预警。
- 预警只说明风险，不改变切线位置或吸附玩家手势。

### 4.3 命中瞬间

- 成功切割确认后产生 30 至 50 毫秒的视觉停顿。
- 停顿只冻结表现层，不丢失指针事件，不改变物理时间步。
- 同一时刻出现银白切光和轻微画面震动。

### 4.4 墨板断开

- 被切除区域沿切线法线方向产生明确位移。
- 位移必须让玩家看出哪一块被切掉，不能仅降低透明度。
- 保留区域、墨刃和墨铁封边继续遵守现有切割规则。

### 4.5 释放

- 墨雾主要向被切除方向爆散，粒子数量随切除面积变化。
- 移动端在允许时触发 10 至 20 毫秒轻振动。
- 单次切割不使用语音，避免打断节奏或产生疲劳。

### 4.6 收尾

- 进度条在切割成功后立即按剩余面积同步缩短。
- 音频由锋利前层、材质中层和低频后层组成。
- 切除动画结束后恢复正常物理与输入节奏。

## 5. 切割声音结构

成功切割使用三层声音，不依赖单个大音量样本：

1. 前层：短而锋利的“唰”，提供速度感。
2. 中层：纸板或墨块断开的“嚓”，提供材质感。
3. 后层：短促低频“咚”，提供重量和释放感。

三层必须在切割确认后的同一时间窗内启动。低频不得盖过锋利前层，整体响度应高于普通界面音，但低于系统最大安全响度。墨铁封边碰撞继续使用独立金属声，不与成功切割混用。

## 6. 评分体系

### 6.1 核心规则

- 切割次数不限，不参与扣分。
- 时间决定基础分。
- 剩余生命扣除分数并限制最高评价。
- 三命耗尽是失败，不进入五档通关评价。

### 6.2 计时

- 计时从玩家第一次起笔开始。
- 暂停、设置菜单、浏览器失焦和结算动画不计时。
- 每关配置极速时间、标准时间和宽松时间。

### 6.3 时间基础分

| 完成时间 | 基础分 |
| --- | ---: |
| 小于或等于极速时间 | 100 |
| 极速时间至标准时间 | 由 99 平滑递减至 75 |
| 标准时间至宽松时间 | 由 74 平滑递减至 40 |
| 超过宽松时间 | 由 39 缓慢递减，最低 20 |

同一时间区间使用线性插值，避免跨越一毫秒导致评价大幅跳变。

### 6.4 生命修正和限级

| 剩余生命 | 分数修正 | 最高评价 |
| --- | ---: | --- |
| 3 | 0 | 夯爆了 |
| 2 | -10 | 顶尖 |
| 1 | -20 | 人上人 |
| 0 | 失败 | 无通关评价 |

最终分等于时间基础分加生命修正，再限制在 20 至 100。之后应用生命最高评价限制。

### 6.5 五档评价

| 分数 | 评价 |
| --- | --- |
| 90–100 | 夯爆了 |
| 75–89 | 顶尖 |
| 60–74 | 人上人 |
| 40–59 | NPC |
| 20–39 | 拉完了 |

五档都代表已经通关。“拉完了”是低分通关，不等于三命耗尽。

## 7. 运营校准目标

新玩家完成关卡后的长期目标分布：

| 评价 | 目标占比 |
| --- | ---: |
| 夯爆了 | 5%–10% |
| 顶尖 | 15%–25% |
| 人上人 | 35%–40% |
| NPC | 20%–30% |
| 拉完了 | 5%–10% |

第一版时间参数根据内部试玩设置。后续根据每关完成时间中位数、生命损失和重试率调整，优先保证“人上人”是最常见的首次通关结果。“夯爆了”保持稀有，“拉完了”不得高频出现。

## 8. 统一声音人格

五档评价全部由同一个角色演绎，形成稳定声音品牌。角色的音色、口音和说话习惯保持一致，只改变情绪强度：

- 夯爆了：最大幅度蓄势与爆发，允许拆成“夯——爆——了”三拍。
- 顶尖：肯定、利落、带明显上扬。
- 人上人：得意、认可，但不压过最高档。
- NPC：平淡、略带调侃。
- 拉完了：嫌弃、泄气，但不辱骂玩家。

每档准备 2 至 3 个同角色语气变体并随机播放。所有语音必须自行录制、合法生成或获得明确授权，不直接复制其他游戏素材。

## 9. 魔性结算时间轴

| 时间 | 事件 |
| --- | --- |
| 0–150ms | 最终切割停顿、切光和爆墨完整播放 |
| 150–500ms | 墨雾散开，背景音短暂收束 |
| 500–850ms | 评价文字蓄势，玩家不可误触下一关 |
| 850ms | 评价大字落下，同时启动对应语音 |
| 1000–1400ms | 朱印落下并与盖章音对齐 |
| 1400ms 后 | 显示统计和操作按钮，可正常交互 |

结算允许玩家点击跳过非关键等待，但第一次点击只完成动画，不应误触“下一关”。

## 10. 五档视觉强度

| 评价 | 主要表现 |
| --- | --- |
| 夯爆了 | 最强停顿、朱砂爆墨、双段震屏、完整魔性语音 |
| 顶尖 | 银白切光、朱印冲击、明显语音反馈 |
| 人上人 | 墨波扩散、中等语音与轻震 |
| NPC | 克制落字、平淡语气、弱反馈 |
| 拉完了 | 文字轻微下坠、墨迹漏气式散开、嫌弃语气 |

所有档位使用同一视觉系统，不为每档生成互不相关的贴图。差异通过动画参数、颜色比例、震动强度和语音表演形成。

## 11. 结算界面信息

结算画面按以下顺序展示：

1. 五档评价大字。
2. 关卡编号。
3. 本关用时。
4. 剩余生命。
5. 挥墨次数，仅作趣味统计。
6. 本关最佳用时。
7. 距离下一档需要提升的时间；已经最高档时显示“本关封神”。
8. 再来一次。
9. 下一关。

评价和核心数据必须在手机首屏内完整显示，不依赖滚动。

## 12. 数据与模块边界

实现时拆分为四个独立单元：

1. `CutFeedbackController`：管理成功切割的停顿、震屏、位移、粒子和进度同步。
2. `LevelTimer`：管理首次起笔、暂停、恢复和最终用时。
3. `ResultScoring`：纯函数计算时间分、生命修正、限级和下一档差距。
4. `ResultSequence`：管理结算时间轴、评价视觉、语音、朱印和按钮解锁。

关卡数据增加三个时间参数。评分模块不得直接操作 DOM、Canvas、音频或存档，方便独立测试。最佳用时按关卡存入本地进度数据，旧存档缺少该字段时按空纪录处理。

## 13. 异常与降级

- 浏览器未解锁音频时，完整播放视觉结算，不阻塞下一关。
- 设备不支持振动时静默跳过。
- 页面失焦时暂停计时和结算时间轴，恢复后继续。
- 动画降帧时优先保留切割判定、切光、进度和评价文字，减少粒子数量。
- 用户关闭音效时不播放语音和音效，但保留全部视觉信息。
- `prefers-reduced-motion` 开启时降低震屏和大幅位移，不删除评分信息。

## 14. 验收标准

### 14.1 爽切反馈

- 成功切割确认、切光、声音和进度更新的感知延迟低于 100 毫秒。
- 切除区域方向明确，不出现原地淡出的假切割感。
- 连续快速切割不会导致输入丢失、音频无限叠加或画面永久停顿。
- 手机振动和震屏均不影响切线命中精度。

### 14.2 评分

- 相同关卡、时间和生命必定得到相同结果。
- 切割次数变化不改变最终分数。
- 剩余两命不能获得“夯爆了”，剩余一命不能高于“人上人”。
- 三命耗尽进入失败流程，不出现五档通关评价。

### 14.3 结算

- 最终切割完整展示后才出现评价。
- 评价文字和对应语音在同一时间窗启动。
- 朱印接触画面时播放盖章声。
- 快速点击只能跳过动画，不能误触下一关。
- 静音、失焦、低性能和减少动态效果状态下仍能完成结算。

## 15. 实施顺序

1. 先实现计时、评分纯函数和测试。
2. 再实现结算界面数据与基础时间轴。
3. 接入现有朱印和临时语音占位，校准同步关系。
4. 升级成功切割的停顿、位移、粒子、震动和三层声音。
5. 导入统一声音人格的正式五档语音。
6. 用 15 关实际试玩数据校准每关时间标准和评价分布。
