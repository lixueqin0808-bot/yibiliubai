# 一笔留白十五关地图定稿设计

## 目标

一次性完成十五关地图迁移，让每关在手机画面中有清楚区别，同时保持现有切割、反弹、封边、难度与美术系统稳定。首版优先完整和可靠，不再为单张复杂造型反复返工。

## 核心原则

- 十五关使用十五张不同的实际多边形，但不强求十五种复杂物件。
- 轮廓负责玩法和第一层识别，低对比纹样负责第二层识别。
- 禁止洞、断开区域、窄连接颈、深凹口和小于墨刃直径的通道。
- 更换地图后，只允许调整墨刃出生坐标和封边索引，不改变数量、速度、种类和目标比例。
- 所有纹样都裁切在当前保留区域内，地图被切掉时纹样同步消失。
- 不新增图片素材；地图轮廓、刻线和纹样均由代码生成。

## 基础图形使用规则

三角形属于稳定凸多边形，可直接进入第 3 关。H 型只采用宽通道版本：左右立柱和中间横桥的有效宽度必须大于最大墨刃碰撞直径加安全余量。心形只保留浅顶部凹口，禁止尖锐深槽。

五角星不作为首版外轮廓。真实星形有五个深凹角，容易形成墨刃卡点；第 7 关继续使用近圆星盘，并通过内部五角星放射刻线建立识别。

## 内部纹样

纹样由 `shapeId` 决定，并统一在地图表面纹理之后、光影覆盖之前绘制：

- 星盘：中心五角星与五条短放射线。
- 叶片：一条主脉与四条弧形侧脉。
- 六边玉牌：一圈低对比内边框。
- 竹简：两道横向竹节线。
- 灯笼：五条纵向骨线与上下两道箍线。
- 八卦：靠近八条边的八组简化卦纹，不绘制两个圆点或圆洞。

纹样透明度保持在 12% 至 18%，不得降低墨刃和封边辨识度。

## 八卦阵设计

第 15 关使用规则八边形，不包含孔洞。八组卦纹沿八个方向排列，每组由三条短线组成；断爻通过中间留缝表达。中央不绘制阴阳双点，避免误读为障碍物或可收集物。

八卦识别依赖“八边外轮廓 + 八组卦纹”，而不是复杂中心图案。

## 圆弧策略

首版继续使用 10 至 16 段均匀线段近似圆弧，应用于星盘、水滴、叶片、心形和灯笼。当前碰撞、切割、侧壁与封边共享同一多边形，因此画面和实际边界不会错位。

十五关完成后单独启动“圆弧渲染升级”：

1. 用线段、圆弧和二次曲线组成初始轮廓描述。
2. 物理与切割继续消费统一采样后的多边形。
3. 渲染层根据原始曲线绘制平滑顶面与外轮廓。
4. 验证第一次切割后原始曲边与新切边能够同时保留，不发生视觉跳变。

本轮不实现真实曲线碰撞、曲线切割、洞或断开区域。

## 数据与代码边界

- `mapShapes.ts` 只生成和验证十五种地图多边形。
- `createLevel.ts` 继续把形状、墨刃和封边索引组装为现有 `LevelDefinition`。
- `goldenLevel.ts` 只声明关卡数据，不再保留迁移关卡的内联多边形。
- `Game.ts` 的地图纹样绘制改为按 `shapeId` 分派的小型独立函数，删除折扇专用代码。

## 验证标准

- 十五关全部声明 `shapeId`，且十五张实际多边形互不相同。
- 所有地图简单、顺时针、无自交，面积和边长位于安全范围。
- 所有墨刃出生在地图内部，具备碰撞半径加 2 像素的边缘余量，彼此不重叠。
- H 型凹槽和心形凹口不会卡住墨刃。
- 所有封边覆盖初始地图的一整条真实边，并随切除区域消失。
- 390 x 844 手机视口与桌面居中视口无溢出。
- `npm.cmd run build`、`npm.cmd run test`、`npm.cmd run test:e2e` 全部通过。

## 本轮不做

- 折扇、中国结、葫芦、梅花和真实凹形五角星。
- 定时炸弹墨刃和其他墨刃特性。
- 新图片素材、音效重调和整体美术重制。
- 真正曲线碰撞与曲线切割。
