# 客观评分再校准设计

## 目标

让“夯爆了、顶尖、人上人、NPC、拉完了”五档评价都能在正常游玩中出现，同时保留无限挥墨的解压体验。评分只看完成用时和剩余墨点，不把挥墨次数、无效切割次数纳入扣分。

首轮目标分布：夯爆了 5%-10%，顶尖 20%-25%，人上人 35%-40%，NPC 20%-25%，拉完了 5%-10%。这不是强制随机分配，而是十五关时间基准校准后的预期自然分布。

## 评分模型

时间基础分保持连续变化：

- 用时不超过 `fastMs`：100 分。
- `fastMs` 到 `standardMs`：从 99 分线性下降到 75 分。
- `standardMs` 到 `relaxedMs`：从 74 分线性下降到 40 分。
- 超过 `relaxedMs`：从 39 分继续下降，最低 20 分。

剩余墨点修正：

- 剩余 3 枚：不扣分，最高“夯爆了”。
- 剩余 2 枚：扣 12 分，最高“顶尖”。
- 剩余 1 枚：扣 28 分，最高“人上人”。
- 0 枚：本关失败，不进入结算评价。

最终档位：

- 90-100：夯爆了。
- 75-89：顶尖。
- 60-74：人上人。
- 40-59：NPC。
- 20-39：拉完了。

这一组合保证：一命通关即使速度极快也只能到“人上人”；一命且普通速度会进入“NPC”；一命且明显偏慢会进入“拉完了”。两命通关仍有机会获得“顶尖”，但普通速度更容易落到“人上人/NPC”。

## 结算解释

结算页继续显示分数、用时、剩余墨点、挥墨次数和最佳时间。保留“再快 X 秒可至下一档”的提示，让玩家知道提升方向；不在教程中公开完整公式，避免玩家只盯着算分。

“点击跳过动画”保持为隐式防误触能力：结算动画尚未完成时，第一次点击只会立即展示完整结算，不能触发下一关、重玩或选关。界面不增加单独的“跳过”按钮。

评价金字与“留白已成”朱印之间保留约 10 像素的视觉间距，不再上下贴合；只调整排版，不改变现有字号、印章尺寸和动画节奏。

## 验证

自动测试覆盖五档边界、三种剩余墨点、切割次数不影响评分，以及结算按钮防误触。人工验收时至少完成十五关一轮，并记录每档出现次数。若前三档仍超过总结果的 80%，下一轮只调整各关时间基准，不改变评分公式和档位文案。

## 发布边界

评分校准在完整 V2 项目中完成并通过构建、单元测试和浏览器测试后，再同步到连接 GitHub 的本地仓库。发布前必须确认 Pages 入口指向 V2 构建产物，不能覆盖或丢失旧版源文件；公网部署成功后再进行手机端测试。

当前 `v2` 目录本身不是 Git 仓库，因此本规格不声称已经提交或发布。
