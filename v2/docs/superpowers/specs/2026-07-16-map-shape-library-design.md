# 多形状地图系统设计

## 目标

在冻结现有美术、玩法和音效的前提下，为十五关建立十五种可明显区分的地图轮廓，使当前版本从“玩法已经完整但地图重复”推进到“内容完整、可测试、可发布”的精品 MVP。

本轮优先级是完整度和稳定性，不追求一次性实现真正曲线物理，也不重新生成地图、墨刃、边条或界面素材。

## 已确认策略

- 地图继续使用简单多边形作为唯一视觉与物理边界。
- 圆形、扇形、叶片和水滴使用 12 至 24 个顶点模拟曲线。
- 继续复用当前切割、面积、反弹、外扩侧墙、阴影和墨铁封边系统。
- 每个轮廓必须是单一闭合区域，不能包含孔洞、断开的组成部分或自相交边。
- 本轮不实现爱心深凹口、五角星、中国结和竹节组合体。这些形状容易让一条切线与地图产生四个以上交点，而当前切割算法只接受两个交点。
- 本轮不增加新图片，不修改当前色彩体系。

## 三种技术路线

### 路线 A：多边形模拟曲线（采用）

所有地图最终都是 `Point[]`。直边形状直接定义顶点；圆弧形状通过采样函数生成顶点。优点是对现有系统改动最小，性能稳定，测试成本最低。

### 路线 B：视觉曲线加多边形物理

Canvas 使用贝塞尔曲线绘制，碰撞仍使用多边形。视觉更平滑，但画面边界和物理边界可能不一致，不适合当前强调切割准确性的版本。

### 路线 C：完整曲线路径

切割、碰撞、侧墙和封边全部理解曲线。效果上限最高，但会把地图扩展变成底层重构，超出当前小游戏的成本边界。

## 模块边界

### 地图形状库

新增 `src/levels/mapShapes.ts`，只负责根据形状标识生成多边形：

- 暴露十五个稳定的 `MapShapeId`。
- 暴露 `createMapShape(shapeId)`。
- 内部提供圆弧采样、点集平移、顺时针统一和重复点清理。
- 生成结果限定在逻辑画布 `390 x 844` 的地图安全区域内。

### 关卡工厂

新增 `src/levels/createLevel.ts`，把地图形状、墨刃、目标比例和封边索引组装成现有 `LevelDefinition`。

封边不再手工复制坐标，改为使用 `lockedEdgeIndices` 指向初始地图的整条边。工厂负责把索引转换成现有 `BoundarySegment[]`，因此 `Game`、切割和渲染无需理解新的封边格式。

### 关卡数据

`src/levels/goldenLevel.ts` 继续作为十五关唯一入口，但每关只声明：

- 形状标识
- 目标比例
- 墨刃配置
- 封边索引
- 第一关引导线

不在关卡文件内继续维护大段重复多边形坐标。

## 几何约束

- 每张地图 3 至 24 个顶点。
- 顶点统一为顺时针。
- 相邻点距离不少于 10 个逻辑像素。
- 地图不得自相交。
- 初始面积保持在统一安全区间，避免难度仅由面积突变造成。
- 圆弧采样不超过 24 个顶点，防止侧墙出现过密接缝。
- 地图必须留在顶部进度条和底部控制按钮之间。

## 墨刃出生安全

每关必须通过自动验证：

- 墨刃中心位于地图内部。
- 墨刃中心到地图边缘的距离大于碰撞半径加 2 像素。
- 墨刃之间不重叠。
- 墨刃不出生在墨铁封边内部。

如果更换形状后旧坐标不再安全，只调整该关墨刃出生坐标，不改变其数量、速度和类型，避免同时重做难度。

## 难度原则

地图形状负责改变反弹方向和可切路径，不负责单独制造难度跳跃。十五关继续沿用当前总体递增的目标比例、墨刃数量、速度和封边数量。

每关最多突出一个主要压力：斜边、窄空间、圆弧反弹、墨刃数量、速度或封边路线。不得在同一关同时大幅提高四项压力。

## 分批交付

### 第一批：六张代表地图

实现梯形、盾牌、菱形、圆形、扇形和叶片，覆盖直边、尖角、斜边和近似圆弧。通过自动测试和浏览器截图后，再扩展全量。

### 第二批：十五关全量

完成剩余九张轮廓，调整不安全的墨刃出生点和封边索引，执行十五关连续截图测试。

### 第三批：完整性收尾

只修复影响游玩的几何、难度和移动端问题，更新说明文档并形成可发布版本。不在此阶段重新讨论整体美术方向。

## 验收标准

- 十五关轮廓在缩略截图中可以明显区分。
- 六类代表几何都能稳定切割和反弹。
- 不出现自相交、无效面积、墨刃出生越界或封边悬空。
- 每次切割后，侧墙和阴影按新多边形正常重建。
- 墨铁封边始终覆盖初始地图的一整条真实边，并随被切除区域消失。
- `npm.cmd run build`、`npm.cmd run test`、`npm.cmd run test:e2e` 全部通过。
- 390 x 844 手机视口与桌面居中视口均无溢出。

## 后续玩法路线图（本轮不实现）

地图系统稳定后，再单独设计“墨刃特性系统”。第一优先级候选是定时炸弹墨刃：出生时携带明确倒计时，8 秒后在原位置分裂为三个小型四锋墨刃。

该机制需要独立解决倒计时可读性、分裂前预警、三个子墨刃的安全出生方向、瞬间重叠处理、音效和性能，因此不能作为地图形状任务的附带修改。

后续可扩展的特性包括：高速、重型、变速、短暂停顿和分裂。每种特性必须有独立视觉语言，玩家无需阅读文字即可判断。

## 范围外

- 真正曲线碰撞与切割
- 带孔洞或断开结构的地图
- 新地图材质和新背景
- 墨刃美术重制
- 新音效与音量调整
- 道具、Boss、养成和商业系统
- 定时炸弹墨刃及其他墨刃特性
