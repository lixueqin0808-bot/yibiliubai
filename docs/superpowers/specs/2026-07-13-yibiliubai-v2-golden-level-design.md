# 一笔留白 V2：黄金样板关设计方案

日期：2026-07-13  
状态：已确认方向，等待用户审阅书面方案

## 1. 项目目标

重制“一笔留白”，以初代 iSlash 的核心切割体验为玩法蓝本，建立一套原创的手机竖屏水墨游戏。第一阶段只完成一个“黄金样板关”，依次验收切割准确性、碰撞规则、反馈爽感、视觉方向和设备表现。样板关通过后，才使用同一套系统扩展到五关。

本阶段不追求内容数量。目标是证明核心循环已经达到可复用、可扩展和适合公开展示的质量。

## 2. 产品边界

- 产品名称暂时沿用“一笔留白”。
- 第一目标平台是手机竖屏网页。
- 游戏使用电脑开发，并支持鼠标完整操作和调试。
- 成品通过 GitHub Pages 上线，可通过 iframe 或网站组件容器嵌入个人网站。
- 后续可以适配为小红书小工具，但第一阶段不处理平台专属格式。
- 第一阶段不加入 Boss、锦鲤、生命值、切割次数限制、奖励道具、任务、养成、多人模式和商业系统。
- 旧版保留作为历史对照，不在旧结构上继续堆叠 V2 功能。

## 3. 原创与参考边界

V2 参考初代 iSlash 的成熟玩法结构：玩家切除没有移动危险物的区域，在更小的空间内继续切割，达到目标比例后过关。

以下内容全部原创：

- 产品名称、世界观和文案
- 水墨视觉、角色造型和动画
- 关卡地图和参数
- UI 布局和交互表现
- 音效、音乐和反馈节奏

不提取、不临摹和不复用初代游戏的素材文件、关卡数据、界面、名称、音频或代码。

## 4. 设计支柱

### 4.1 准

切线显示、碰撞结果和区域清除必须与玩家看到的画面一致。不能出现“看起来没碰到却失败”或“明明穿过画面却没有切开”的情况。

### 4.2 爽

拖动阶段克制、安静；成功瞬间锋利、短促、有重量。爽感来自视觉、声音、轻震、节奏停顿和面积变化共同完成，不能只依赖粒子数量。

### 4.3 清

玩家第一次进入游戏后，应在 30 秒内通过互动理解三个事实：从边缘切到边缘、不要碰墨刃、切除没有墨刃的一侧。

### 4.4 轻

游戏需要适合 GitHub Pages、个人网站嵌入和手机浏览器。第一阶段不依赖视频背景或大型游戏引擎，控制资源体积和启动时间。

## 5. 黄金样板关

### 5.1 关卡定义

- 关卡名：初试锋芒
- 画幅：390 x 844 逻辑尺寸，按设备比例缩放
- 可切区域：原创的不规则纵向画卷轮廓，不使用简单矩形
- 危险物：一个普通旋转墨刃
- 目标：清除 50% 的初始可切面积
- 墨刃速度：教学阶段偏慢，完成第一笔后恢复标准速度
- 切割次数：不限
- 失败代价：本关立即快速重开，不扣除资源
- 评价：达到目标即可过关；完成时间和有效切割次数共同计算一至三星

### 5.2 核心流程

1. 玩家首次进入时，游戏直接显示样板关，不经过营销页。
2. 画面用动态示范线提示第一笔的起点和方向。
3. 玩家按住并拖动时，系统实时显示从起点到当前位置的直线笔锋。
4. 松手后，系统判定切线、墨刃和边界关系。
5. 成功时，保留含墨刃的一侧，清除没有墨刃的一侧。
6. 失败时，显示碰撞位置并快速重开本关。
7. 留白达到 50% 后进入结算，显示星级、时间和有效切割次数。

## 6. 切割规则

### 6.1 有效切割

- 手势起点和终点都必须落在画卷边界附近的有效容差范围内。
- 切线必须与当前可切多边形形成两个有效交点。
- 物理切割使用起点和终点构成的直线段；视觉笔锋可以表现速度和压力变化，但不得改变真实碰撞位置。
- 切线与墨刃碰撞时，整次切割失败并触发本关重开。
- 所有墨刃都在同一侧时，保留该侧并清除另一侧。
- 两侧都含墨刃时，不改变区域，给予短促的无效切割反馈。

### 6.2 面积与完成条件

- 进度按当前可切面积相对于初始面积的减少比例计算。
- 数值显示取整，但内部计算保留精度。
- 达到或超过 50% 时立即完成关卡。
- 星级由配置化阈值计算，避免写死在界面逻辑中。

### 6.3 后续机制接口

黄金样板关只实现普通墨刃，但数据结构需要预留以下类型：高速墨刃、大型慢速墨刃、幽灵墨刃和不可切割边界。接口预留不等于提前实现这些玩法。

## 7. 反馈设计

### 7.1 起笔与拖动

- 按下时播放轻微笔触声。
- 笔锋从接触点出现，拖动速度影响视觉宽度和飞白程度。
- 危险接近只通过笔锋轻微泛朱和墨刃呼吸变化提示，不使用持续的大红警告圈。
- 浏览器页面在操作期间不得滚动、缩放或触发长按菜单。

### 7.2 成功切割

- 松手后约 40 毫秒节奏停顿。
- 切线闪出锋利高光，纸面沿切线分离。
- 被清除区域轻微错位，并在约 350 毫秒内化为墨雾和飞散颗粒。
- 使用短促切纸声、墨散声和低频落点声构成三层音效。
- 支持设备时触发一次轻震。
- 进度数字和目标标记同步更新，不等待动画完全结束。

### 7.3 碰撞失败

- 碰撞瞬间冻结约 60 毫秒。
- 精确碰撞点爆开少量朱红墨迹。
- 播放短促断弦或裂笔声，并触发一次轻震。
- 使用约 250 毫秒的水墨遮罩恢复关卡。
- 从碰撞到重新可操作不超过 0.6 秒。

### 7.4 无效切割

当切线没有完整穿过区域，或切割后两侧都含墨刃时，不重开关卡。笔锋快速变淡，并用一条不超过八个字的提示说明原因。

## 8. 视觉设计

### 8.1 方向

整体采用高对比黑白泼墨、锋利笔触和宣纸扩散质感，少量朱红只用于危险、失败和印章。首版不加入彩色水墨，避免视觉语言失焦。

### 8.2 移动危险物

基础危险物统一为“旋转墨刃”。它保留清楚的圆形碰撞轮廓，但外观由不规则笔锋组成。后续不同能力通过笔锋数量、墨色浓淡、旋转速度和残影区分。

### 8.3 素材分工

AI 生成并由人工挑选的素材：

- 宣纸与泼墨背景
- 墨刃透明素材及视觉变体
- 金属边材质
- 章节题字和朱红印章
- 墨雾、飞白和散墨纹理

代码实时生成的内容：

- 关卡形状和精确碰撞遮罩
- 墨刃移动、旋转和残影
- 切线、接近危险和碰撞定位
- 粒子、分离、淡出和屏幕轻震
- UI 状态和响应式布局

AI 图片只提供外观和纹理，碰撞轮廓始终由游戏数据控制，防止素材边缘与实际判定错位。

### 8.4 动画边界

黄金样板关不使用生成视频。五关完成后，再评估是否制作 5 至 8 秒的开场水墨动画或章节转场。核心玩法不能依赖视频才能成立。

## 9. 界面与基础功能

### 9.1 游戏界面

- 顶部：产品名、关卡名、留白进度、暂停和声音图标
- 中部：占据主要视口的游戏画卷
- 底部：只显示当前必要提示，不长期堆放按钮
- 桌面端：竖屏游戏居中显示，鼠标操作规则与触屏一致

### 9.2 教程

教程嵌入第一关，不先展示长篇说明：

1. 动态示范线引导第一笔。
2. 首次成功后表现“含墨刃的一侧保留”。
3. 墨刃靠近笔锋时出现一次危险提示。

帮助页面提供完整规则、元素图例和失败原因，玩家可随时重新查看。

### 9.3 必备状态

- 加载状态
- 首次开始状态
- 游戏中状态
- 暂停状态
- 成功结算状态
- 失败快速重开状态
- 帮助状态
- 静音状态

暂停页提供继续、重试、帮助、声音和返回选关。黄金样板关阶段选关只有第一关入口，扩展五关后再启用完整关卡列表。

## 10. 技术方案

### 10.1 路线

采用 TypeScript、Canvas 2D 和 Vite。移动物理使用 Matter.js，区域运算使用经过测试的多边形几何模块。V2 不使用 Phaser 等完整游戏引擎，以降低嵌入体积和不必要的抽象。

### 10.2 模块边界

```text
v2/
  src/
    core/       游戏状态、时钟和主循环
    geometry/   切线、多边形、面积和区域判断
    physics/    墨刃运动、边界碰撞和同步
    render/     画卷、素材、笔锋和粒子
    levels/     关卡配置和机制类型
    audio/      音效、静音和浏览器解锁
    ui/         教程、暂停、帮助和结算
  assets/
    images/
    textures/
    audio/
  tests/
```

核心模块通过明确数据接口通信。物理层只提供位置和碰撞事件；几何层只处理切割结果；渲染层不决定游戏规则；UI 层不直接修改物理对象。

### 10.3 嵌入与发布

- 构建结果是可托管在 GitHub Pages 的纯静态文件。
- 页面支持独立访问和 iframe 嵌入。
- 游戏监听容器尺寸变化，不假设全屏运行。
- 对外预留暂停、继续、重试和静音接口。
- 进度保存在 localStorage；无法访问存储时游戏仍可正常运行，只是不保留记录。

## 11. 测试与验收

### 11.1 自动测试

- 任意角度切线与多边形产生正确交点
- 极小切片不会产生无效多边形
- 面积守恒误差处于允许范围
- 墨刃位于不同侧时不错误清除区域
- 切线和圆形墨刃的碰撞判定覆盖高速移动情况
- 星级阈值和进度存储结果正确

### 11.2 设备验收

- 390 x 844 逻辑画布在常见手机竖屏下完整显示
- 手机触控、电脑鼠标均可完成关卡
- 操作时页面不滚动、不缩放、不触发长按菜单
- 主流手机运行时接近 60 帧
- 首屏资源尽量控制在 3 MB 以内
- 点击或触摸到画面反馈尽量控制在 50 毫秒内
- 碰撞失败后 0.6 秒内恢复操作
- 初次玩家在 30 秒内理解核心规则
- GitHub Pages 独立访问和个人网站嵌入均正常

### 11.3 质量门槛

只有以下五项依次通过，才能扩展到五关：

1. 切割准确性：视觉线、碰撞线和切割结果一致。
2. 碰撞规则：不同速度、角度和边界情况下结果稳定。
3. 反馈爽感：成功与失败都清楚、有节奏且不拖沓。
4. 视觉方向：AI 素材与程序动画形成统一水墨语言。
5. 设备表现：手机竖屏、电脑操作、独立页面和嵌入模式均通过。

## 12. 协作流程

### 12.1 Codex 负责

- 正式规格、实施计划和代码
- 几何、物理、渲染、UI、音效和测试
- AI 美术提示词与素材技术规格
- 素材裁切、压缩、透明化、对齐和集成
- 本地、Obsidian、GitHub 和 GitHub Pages 同步验证

### 12.2 用户负责

- 在两个明确的视觉候选方案中做选择
- 按 Codex 提供的最终提示词使用 Image 2 生成指定素材
- 试玩黄金样板关并评价切割是否准确、是否解压、是否有反馈感
- 五关完成后，根据需要使用即梦生成开场或章节动画

### 12.3 确认节点

1. 视觉提示词和两套风格候选确认
2. 无美术占位版的切割与碰撞验收
3. 黄金样板关完整美术与反馈验收
4. 扩展五关前的质量门槛确认

每个节点都先给用户可直接体验或选择的结果。只有必须依赖用户审美判断或外部生成工具时才请求协助，普通实现问题由 Codex 自行解决。

## 13. 后续五关扩展方向

黄金样板关通过后，按以下顺序扩展：

1. 初试锋芒：一个普通墨刃和完整教程
2. 双刃同归：两个普通墨刃
3. 疾徐之间：高速小墨刃与低速大墨刃
4. 铁画银钩：不可切割金属边
5. 墨影无踪：普通、高速和幽灵墨刃综合挑战

奖励道具放到五关精品版完成后的下一阶段，不与黄金样板关并行开发。
